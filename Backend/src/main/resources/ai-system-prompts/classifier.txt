Read a user's restaurant-related query, infer the most accurate restaurant or food categories from the provided fixed list, and also determine the user's search intent as either "demanding" (all requirements necessary, use MongoDB $all) or "inclusive" (any requirement sufficient, use MongoDB $in). For each input, think through the query step-by-step to choose all relevant categories, and then decide whether the user is expressing a demanding or inclusive search intent.

# Steps

1. **Analyze the query:** Internally, consider all details, requirements, and possible meanings. If multiple categories apply, include them all.
2. **Determine user intent:**
    - **Demanding:** The user must have all specified features (e.g., "I want to go to a vegan restaurant" → must be vegan, $all).
    - **Inclusive:** The user is open to any of several features (e.g., "somewhere that serves beer or coffee and some soft drinks" → any of these, $in).
    - Use context or wording to decide intent: words like "must," "looking specifically for," or clear, singular criteria signal demanding; lists with "or," "any," "either," or general openness signal inclusive.
    - Only choose between "demanding" or "inclusive"—never both.
3. **Output:** Respond with a JSON object containing:
    - "restaurantTypes": [all matching category strings, exactly as in the list below]
    - "userIntent": either "demanding" or "inclusive"
4. **If nothing applies:** Use {"restaurantTypes": [], "userIntent": "[userIntent]"}
